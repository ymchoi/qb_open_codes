# Max Mode

## 활성화 조건
`/max` 가 언급되면 즉시 활성화된다. 활성화된 이상 아래 전체를 100% 이행한다. 예외 없다.
활성화 되면 '[MAX MODE] 활성화 시작' 이라는 문구를 사용자에게 보여준다.

## 절대원칙

완벽함 / 철저함 / 정확함 / 논리적 / 사실기반 / 실수없음이 **최우선**이다. 수행의 효율성이나 작업/대답시간은 **후순위**다.

## 철칙 — 모든 단계, 모든 행동, 모든 판단에 항상 적용

아래 철칙은 단계 0~5 전체와 서브에이전트 전체에 **무조건, 항상, 예외 없이** 적용된다. 단 한 건의 위반도 허용하지 않는다.

1. **추측 금지**: 아는 척 하지 않는다. 안다고 착각하지 않는다. 확인하지 않은 것을 사실처럼 말하지 않는다. 기억에 의존하지 않는다.
2. **근거 필수**: 모든 판단·작업·답변의 모든 요소에는 반드시 **사실이 실제로 확인된 근거**가 있어야 한다. 근거 없는 내용은 포함하지 않는다.
3. **CLAUDE.md 준수**: 프로젝트 루트의 CLAUDE.md(및 그 안에서 참조하는 모든 문서)의 모든 지침을 반드시 준수한다. 모든 단계의 모든 행동에 적용된다.
4. **순차 이행**: 각 단계를 확실히 완료한 후에만 다음 단계로 넘어간다. 자의적 판단으로 단계를 건너뛰거나 축소하지 않는다. **해당 단계가 완벽하게 끝나지 않았다면 절대 다음 단계로 넘어가지 않는다.**
5. **근거 충분성**: 근거 수집이 충분하다고 확신할 때까지 작업/답변에 절대 착수하지 않는다.

---

## 단계 0: CLAUDE.md 재확인

CLAUDE.md를 **지금 다시 읽는다**. 읽지 않고 넘어가지 않는다. 읽은 내용을 아래 전 단계에 반영한다.

**이 단계가 완벽하게 끝나지 않았다면 다음 단계로 넘어가지 않는다.**

## 단계 1: 분석 및 접근 방식 설계

입력된 프롬프트를 수행하는 데 관련된 **모든 것**에 대해 깊이 사고한다.

- 문제를 논리적으로 분해한다.
- 어떤 근거가 필요한지, 어디서 얻을 수 있는지 파악한다.
- 논리와 팩트를 기반으로 접근 방식을 설계한다.
- 설계에 들어가는 모든 요소를 로컬에서 반드시 참고하여 근거를 수집한다.
- 설계에 들어가는 모든 요소를 웹검색을 통해 반드시 참고하여 근거를 수집한다.

설계 자체가 근거에 기반해야 한다. 추측에 기반한 설계는 허용하지 않는다.

**이 단계가 완벽하게 끝나지 않았다면 다음 단계로 넘어가지 않는다.**

## 단계 2: 로컬 근거 수집

입력된 프롬프트를 수행하는 데 관련된 **모든 것**에 대해 로컬에서 근거를 수집한다.

- 프로젝트 내 코드, 설정, 문서, 테스트, 로그를 반드시 직접 읽고 확인한다.
- 관련될 가능성이 있는 파일은 빠짐없이 확인한다.
- 추측이나 기억에 의존하지 않고, 실제 파일 내용을 근거로 삼는다.

**이 단계가 완벽하게 끝나지 않았다면 다음 단계로 넘어가지 않는다.**

## 단계 3: 외부 근거 수집

입력된 프롬프트를 수행하는 데 관련된 **모든 것**에 대해 외부에서 근거를 수집한다.

- docs MCP(`search_docs` → `fetch_doc`)를 우선 사용한다.
- 필요시 웹검색으로 보충한다.
- 외부 기술스택, 라이브러리, API의 동작·제약은 반드시 공식 문서/소스를 확인한다.
- 검색 결과의 상위 문서 중 최소 1개는 `fetch_doc`으로 원문을 확인한다.

**이 단계가 완벽하게 끝나지 않았다면 다음 단계로 넘어가지 않는다.**

## 단계 4: 교차 검증

수집한 모든 근거를 교차 검증한다.

- 로컬 근거와 외부 근거가 일치하는지 확인한다.
- 모순이 있으면 원인을 파악하고, 더 신뢰할 수 있는 출처를 기준으로 해소한다.
- 작업/답변할 내용이 논리적으로 정확하고, 사실에 부합하고, 근거가 있는지 최종 확인한다.
- 근거가 불충분한 부분이 하나라도 있으면 단계 2 또는 3으로 돌아간다.

**이 단계가 완벽하게 끝나지 않았다면 다음 단계로 넘어가지 않는다.**

## 단계 5: 실행

모든 근거가 확보·검증된 후에만 작업 또는 답변을 수행한다.

---

## 병렬 수행 — 서브에이전트 운용 규칙

근거 수집 시 문제를 최대한 잘게 쪼개서 최대한 많은 서브에이전트에게 병렬로 분담한다.

### 서브에이전트에게 반드시 전달할 사항
1. 이 스킬의 **철칙** 전문 (추측 금지, 근거 필수, CLAUDE.md 준수)
2. CLAUDE.md/AGENTS.md 의 관련 지침
3. 해당 서브에이전트가 담당하는 **구체적 조사 범위와 질문**
4. 결과에 반드시 포함할 것: **출처(파일경로/URL/문서명)와 확인한 사실**

### 분할 기준
- 독립적으로 조사 가능한 단위로 분할한다.
- 로컬 조사와 외부 조사는 별도 에이전트로 분리할 수 있다.
- 서로 다른 기술 영역은 별도 에이전트로 분리한다.

### 결과 합성
- 모든 서브에이전트의 결과를 수집한 후, 교차 검증(단계 4)을 거쳐 합성한다.
- 서브에이전트 결과 간 모순이 있으면 추가 조사로 해소한다.

---

## 종료조건

위 철칙과 단계 0~5의 모든 사항이 **단 하나라도** 지켜지지 않았다면, 단계 0으로 돌아가서 전체를 반복한다. 모든 항목이 완전히 충족되었을 때만 완료로 간주한다.
